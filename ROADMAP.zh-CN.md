# DeepSeek Session Manager 路线图

本文档用于规划 DeepSeek Session Manager 后续版本升级、项目优化和功能迭代。

项目当前定位是一个本地优先的 Agent 会话管理桌面应用，核心目标不是替代 DeepSeek TUI 或 Claude Code，而是帮助用户更快地找到历史会话、理解会话来源，并一键恢复到对应 CLI 工具中继续工作。

## 核心原则

- 本地优先：会话数据只从本机读取，不上传、不远程同步。
- 只读优先：默认不修改 DeepSeek TUI 和 Claude Code 的原始会话文件。
- 恢复优先：所有功能都应服务于“快速找到并恢复会话”这个核心目标。
- 安全可控：涉及归档、删除、移动文件的功能必须谨慎设计，默认避免破坏性操作。
- 逐步扩展：先把 DeepSeek TUI 和 Claude Code 做稳定，再考虑支持更多 Agent 工具。

## 当前版本：v0.1.x

当前版本主要验证产品方向和基础功能。

已具备的能力：

- 支持读取 DeepSeek TUI 本地会话。
- 支持读取 Claude Code 本地会话。
- 支持在 DeepSeek TUI 和 Claude Code 两种来源之间切换。
- 支持按工作区、时间、模型、模式、收藏等维度查看会话。
- 支持搜索标题、首条用户消息、workspace、模型、模式和会话 ID。
- 支持查看会话详情。
- 支持收藏会话。
- 支持复制恢复命令。
- 支持打开会话文件所在目录。
- 支持从图形界面启动终端并恢复会话。
- 支持 Windows 安装包构建。

v0.1.x 后续只做稳定性修复，不建议加入大功能：

- 修复 DeepSeek TUI 和 Claude Code 启动兼容问题。
- 优化 README、截图、安装说明和常见问题。
- 增加应用内版本号展示。
- 增加 CLI 可用性检测结果展示。
- 优化目录不存在、命令不可用、会话解析失败时的错误提示。
- 补充 GitHub Release 发布说明模板。

## v0.2.0：会话管理增强

这一版重点解决“会话变多以后不好管理”的问题。

计划功能：

- 支持自定义 DeepSeek TUI sessions 目录。
- 支持自定义 Claude Code projects 目录。
- 支持给会话添加本地备注。
- 支持给会话添加本地别名。
- 支持给会话添加标签。
- 支持最近启动记录。
- 支持隐藏或置顶工作区。
- 支持按工作区手动排序。
- 支持导出恢复命令列表。

设计要求：

- 备注、别名、标签、最近启动记录只写入应用自己的状态文件。
- 不修改 DeepSeek TUI 或 Claude Code 的原始会话文件。
- 自定义目录需要支持重置为默认目录。
- 如果目录不可用，需要给出明确提示，而不是静默失败。

建议验收标准：

- 用户可以配置自己的会话目录。
- 用户可以给常用会话添加备注和标签。
- 用户可以快速找到最近恢复过的会话。
- 原始 session 文件不会发生变化。

## v0.3.0：安全归档与清理

这一版开始处理“旧会话太多”的问题，但必须保证安全。

计划功能：

- 支持归档会话。
- 支持查看归档会话。
- 支持从归档恢复会话文件。
- 支持批量归档指定工作区或指定时间之前的会话。
- 支持只读模式和管理模式切换。

设计要求：

- 默认不提供直接删除。
- 归档操作必须二次确认。
- 归档目录应使用应用自己的目录，例如：

```text
%USERPROFILE%\.deepseek-session-manager\archive
```

- 归档前保留必要的元数据，方便恢复。
- 恢复归档时必须避免覆盖同名文件。

建议验收标准：

- 用户可以安全地把旧会话移出主列表。
- 用户可以从归档区恢复会话。
- 即使归档失败，也不会导致原始会话丢失。

## v0.4.0：搜索、筛选和详情优化

这一版重点提升大量会话下的查找效率。

计划功能：

- 支持全文搜索会话标题、首条消息、workspace、模型和路径。
- 支持按时间范围筛选。
- 支持按消息数量筛选。
- 支持按 token 数量筛选。
- 支持按模型筛选。
- 支持按来源筛选。
- 支持会话详情中展示更完整的 Claude Code 信息。
- 支持会话统计面板。

性能优化：

- 增加本地缓存索引，避免每次启动都全量解析所有文件。
- 文件修改后再增量更新缓存。
- 大量会话时保持界面响应流畅。

建议验收标准：

- 几百个会话文件下仍能快速打开应用。
- 搜索结果响应明显快于全量重新扫描。
- 用户能通过时间、模型、工作区快速缩小范围。

## v0.5.0：多工具 Provider 架构

这一版开始为更多 Agent 工具做扩展准备。

计划功能：

- 抽象统一的 `SessionProvider`。
- DeepSeek TUI 作为一个独立 provider。
- Claude Code 作为一个独立 provider。
- 后续可以扩展 Codex、Gemini CLI、Qwen Code 等工具。
- 每个 provider 独立负责扫描、解析、恢复命令和字段映射。

建议架构：

```text
src/
  providers/
    deepseek/
    claude/
  lib/
    search/
    state/
    commands/
  components/
    session-list/
    session-detail/
    sidebar/
    toolbar/

src-tauri/
  src/
    providers/
    commands/
    state/
```

设计要求：

- UI 不直接关心具体 session 文件格式。
- Rust 后端不把所有来源逻辑堆在一个文件里。
- 新增 provider 时，不应大规模改动已有 provider。

建议验收标准：

- DeepSeek 和 Claude 的解析逻辑彼此独立。
- 新增一个 provider 时有明确接口可实现。
- 前端使用统一数据结构渲染不同来源的会话。

## v1.0.0：稳定公开版

这一版目标是让项目具备长期公开发布的基础质量。

计划能力：

- 配置文件格式稳定。
- 支持旧版本状态文件迁移。
- 支持自动更新或至少提供新版本提醒。
- GitHub Releases 提供安装包和更新说明。
- README、中文 README、路线图、常见问题完整。
- 提供明确开源许可证，例如 MIT 或 Apache-2.0。
- 基础测试覆盖核心逻辑。

建议测试覆盖：

- DeepSeek session 解析。
- Claude Code session 解析。
- 会话搜索。
- 会话分组。
- 收藏状态。
- 标签和备注状态。
- 恢复命令生成。
- 状态文件迁移。

建议验收标准：

- 普通用户可以从 GitHub Releases 下载并安装。
- 用户不需要本地开发环境也能使用。
- 应用出现错误时能给出明确提示。
- 更新版本不会破坏已有收藏、备注、标签和设置。

## 技术债整理

后续迭代中建议逐步处理以下技术债：

- 拆分过大的前端组件，降低 `App.tsx` 复杂度。
- 把 DeepSeek 和 Claude 的解析逻辑拆到独立模块。
- 把搜索、分组、排序逻辑独立出来并补充测试。
- 把 Tauri 后端命令按职责拆分。
- 增加错误类型，避免只返回普通字符串。
- 增加日志文件，方便用户反馈问题。
- 增加 UI 空状态、错误状态、加载状态。
- 增加构建和发布 checklist。

## 发布节奏建议

建议采用小版本快速修复、大版本集中交付的节奏：

- `v0.1.x`：只修 bug、文档、发布体验。
- `v0.2.0`：集中交付会话管理增强。
- `v0.3.0`：集中交付安全归档能力。
- `v0.4.0`：集中交付搜索和性能优化。
- `v0.5.0`：集中交付 provider 架构重构。
- `v1.0.0`：公开稳定版。

每次发布建议包含：

- 版本号。
- 新增功能。
- 修复问题。
- 已知限制。
- 安装包下载。
- 升级注意事项。

## 近期优先级

短期最建议先做 `v0.2.0`，因为当前版本已经完成了基础读取、展示和恢复，下一步真正影响使用体验的是会话管理能力。

推荐开发顺序：

1. CLI 可用性检测和错误提示。
2. 自定义会话目录。
3. 最近启动记录。
4. 会话备注和别名。
5. 标签系统。
6. 工作区置顶和隐藏。

这条路线风险较低，且每一步都能单独发布，不需要一次性重构整个项目。

# Project Structure

## Top-Level Layout

```
.
├── src/                        # React + TypeScript frontend
├── src-tauri/                  # Rust + Tauri backend (desktop shell)
├── docs/superpowers/plans/     # Design/implementation plans (markdown)
├── dist/                       # Vite build output (generated — do not edit)
├── node_modules/               # pnpm-managed deps
├── package.json                # Scripts, deps, version source of truth
├── pnpm-lock.yaml              # Canonical lockfile
├── vite.config.ts              # Vite + Vitest config, injects __APP_VERSION__
├── tsconfig.json               # Strict TS config (ES2022, Bundler resolution)
├── tsconfig.node.json          # TS config for Vite-side node code
├── index.html                  # Vite entry (mounts #root)
├── README.md / README.zh-CN.md # User docs (keep both in sync)
├── ROADMAP.zh-CN.md            # Roadmap in Chinese
└── deepseek-session-manager-prd.md   # Original PRD (reference)
```

## Frontend (`src/`)

```
src/
├── main.tsx                    # React root mount
├── App.tsx                     # Top-level UI, state, Tauri invoke calls
├── App.css                     # Desktop styling (light/dark via data-theme)
├── types.ts                    # Shared types: SessionRecord, AppState, GroupBy, SessionSource, DeepseekStatus
├── vite-env.d.ts               # Vite/TS ambient declarations (incl. __APP_VERSION__)
├── assets/                     # Static assets imported by bundler (svg, etc.)
└── lib/                        # Pure helpers — no React, no Tauri IPC
    ├── session.ts              # normalize / match / group / format helpers
    ├── session.test.ts         # Vitest co-located tests
    ├── commands.ts             # buildResumeCommand + CLI name constants
    └── commands.test.ts        # Vitest co-located tests
```

Rules for `src/lib/`:
- Keep modules pure and deterministic. No `invoke`, no DOM, no side effects.
- Every exported function should have a test in the matching `*.test.ts`.
- Types imported from `../types` — do not redefine session shapes here.

Rules for `App.tsx`:
- Owns all IPC calls via `invoke<T>(...)`.
- Handles loading, error, and notice state. Surfaces errors to the UI; never swallow.
- New interactive elements should use the existing `Icon` / `IconButton` pattern.

## Backend (`src-tauri/`)

```
src-tauri/
├── Cargo.toml                  # Rust deps (tauri 2, serde, serde_json, dirs)
├── Cargo.lock                  # Pinned deps
├── build.rs                    # tauri-build codegen hook
├── tauri.conf.json             # App identifier, window size, bundle targets (nsis)
├── capabilities/default.json   # Tauri ACL: allowed commands and scopes
├── src/main.rs                 # All Tauri commands + parsing logic
├── icons/                      # App icons
├── gen/schemas/                # Generated schemas (do not edit by hand)
└── target/                     # Cargo build output (generated)
```

Rules for `main.rs`:
- Every `#[tauri::command]` returns `Result<T, String>` with user-facing Chinese error messages.
- Register new commands in the `invoke_handler![...]` list at the bottom of `main()`.
- Serializable structs use `#[serde(rename_all = "camelCase")]` so the TS side sees camelCase fields.
- Keep parsing (`parse_session_file`, `parse_claude_session_file`) tolerant: a bad file produces an `invalid_record` entry rather than aborting the whole scan.
- Path helpers (`default_sessions_dir`, `default_claude_projects_dir`, `app_state_path`, `home_dir`) are the single source of truth for filesystem locations.

## Documentation (`docs/`)

- `docs/superpowers/plans/` holds dated implementation plans in Markdown. Add new plans here rather than cluttering the repo root.
- User-facing docs live at the repo root (`README*.md`, `ROADMAP*.md`).

## Generated / Ignored

Do not commit or edit by hand:
- `dist/`
- `node_modules/`
- `src-tauri/target/`
- `src-tauri/gen/schemas/` (regenerated by `tauri-build`)

## Where to Add New Code

- **New UI interaction** → extend `src/App.tsx`, reuse `Icon` / `IconButton`.
- **New pure helper** → add to `src/lib/<feature>.ts` with a co-located `.test.ts`.
- **New Tauri command** → add to `src-tauri/src/main.rs`, register in `invoke_handler!`, update `capabilities/default.json` if it needs permissions, and mirror the type on the TS side in `src/types.ts`.
- **New session source** → extend `SessionSource` union in `src/types.ts`, add a parser in `main.rs`, branch in `list_sessions` / `check_agent` / `resume_session`, and extend `buildResumeCommand`.
- **New design/plan doc** → `docs/superpowers/plans/YYYY-MM-DD-<slug>.md`.
